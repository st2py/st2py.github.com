TunSecur Project

 

Published 2015-11-19

TunSecur means Tunnel Security, it is an open source secure tunnel software, consists of client and server, has the following features:

  • Released under The MIT License, fully open source and completely free
  • Developed by Golang, safe, efficient, easy to deploy
  • Support both password and RSA authentication
  • Support RSA 1024, 2048, 4096 key length
  • Support AES 128, 192, 256 key length
  • Support CFB, CTR, OFB iteration mode
  • Support both TCP and UDP protocols
  • Support Linux, Windows, OSX, FreeBSD four main operating systems

Scenario

TunSecur can be used for any scenario where secure network communication is needed, such as:

  • secure DNS Proxy
  • secure SSH Proxy
  • secure VPN Proxy
  • secure Web Proxy

Simple tutorial

The TunSecur network communication involves four endpoints, they are user point, client point, server point and dest point:

  • The user point is the real network communication source, such as web browser, dns client etc.
  • The client point is the tunnel client, used to encrypt data from user point to server point.
  • The server point is the tunnel server, used to decrypt data from client point to dest point.
  • The dest point is the real network communication destination, such as web site, dns server etc.

Command help

Generate RSA key pair

TunSecur will generate public.pem and private.pem files. Public key is in public.pem file, used for tunnel client at RSA mode; private key is in private.pem file, used for tunnel server at RSA mode.

Note: Please backup the key files, authentication will be failed at RSA mode if the key files are lost!

Tunnel Parameters

-ts Set tunnel server mode
-tc Set tunnel client mode

-tcp Set TCP tunnel
-udp Set UDP tunnel

-lp Set listen Host:Port
-remote Set forward Host:Port

-passwd Set password auth mode

-pub Set RSA auth mode, and RSA public file for tunnel client
-priv Set RSA auth mode, and RSA private file for tunnel server

-aes Set AES iteration mode, only valid: ctr, cfb, ofb
-len Set AES key length，only valid: 16, 24, 32

-log Set log bits, default is error bit, set to 7 to open all bit

How it works

Password Auth Mode

At Password Auth Mode，TunSecur client process as follows:

  1. Client and server config the same flag 0x23571719
  2. Client and server config the same password
  3. Client random gens AES key seed (tx_seed_key, rx_seed_key), and init vector seed (tx_seed_iv, rx_seed_iv)
  4. Client encrypts (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) by using AES-CTR-128 and password
  5. Client sends encrypted (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) to server
  6. Client gens AES (tx_key, tx_iv) from (tx_seed_key, tx_seed_iv), uses them to encrypt data to server
  7. Client gens AES (rx_key, rx_iv) from (rx_seed_key, rx_seed_iv), uses them to decrypt data from server

At Password Auth Mode，TunSecur server process as follows:

  1. Server and client config the same flag 0x23571719
  2. Server and client config the same password
  3. Server receives encrypted (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) from client
  4. Server decrypt (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) by using AES-CTR-128 and password
  5. Server checks to see if decrypted flag is equal to the flag; if not authentication is failed, close connection
  6. Server gens AES (tx_key, tx_iv) from (tx_seed_key, tx_seed_iv), uses them to decrypt data from client
  7. Server gens AES (rx_key, rx_iv) from (rx_seed_key, rx_seed_iv), uses them to encrypt data to client

RSA Auth Mode

At RSA Auth Mode，TunSecur client process as follows:

  1. Client and server config the same flag 0x23571719
  2. Client and server config the same password
  3. Client random gens AES key seed (tx_seed_key, rx_seed_key), and init vector seed (tx_seed_iv, rx_seed_iv)
  4. Client encrypts (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) by using RSA and public key file
  5. Client sends encrypted (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) to server
  6. Client gens AES (tx_key, tx_iv) from (tx_seed_key, tx_seed_iv), uses them to encrypt data to server
  7. Client gens AES (rx_key, rx_iv) from (rx_seed_key, rx_seed_iv), uses them to decrypt data from server

At RSA Auth Mode，TunSecur server process as follows:

  1. Server and client config the same flag 0x23571719
  2. Server and client config the same password
  3. Server receives encrypted (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) from client
  4. Server decrypt (flag, tx_seed_key, tx_seed_iv, rx_seed_key, rx_seed_iv) by using RSA and private key file
  5. Server checks to see if decrypted flag is equal to the flag; if not authentication is failed, close connection
  6. Server gens AES (tx_key, tx_iv) from (tx_seed_key, tx_seed_iv), uses them to decrypt data from client
  7. Server gens AES (rx_key, rx_iv) from (rx_seed_key, rx_seed_iv), uses them to encrypt data to client
